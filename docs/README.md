# DyCoV documentation

This directory contains the **main public documentation** for DyCoV.

It is organized by **purpose and audience**, so that users, advanced users,
and developers can easily find the information relevant to them.

Some subdirectories in `docs/` are used for internal structure,
figures, or tooling support and are not intended to be read linearly.
The sections listed below describe the **primary documentation entry points**.

---

## Installation

➡️ `installation/`

Documentation explaining **how to install DyCoV and make it available** on a
system.

This section is intended for **end‑users** and covers:
- installation using the prebuilt distribution image (recommended),
- native Linux installation for advanced users.

Start here if DyCoV is not yet installed.

---

## Tutorials

➡️ `tutorials/`

Step‑by‑step, user‑facing tutorials explaining **how to use DyCoV once it is
installed**.

These tutorials cover the main DyCoV workflows, including:
- RMS model validation,
- electrical performance verification,
- Grid‑Forming (GFM) analysis.

The tutorials are designed to be read in a specific order and form the
recommended learning path for new users.

---

## Developer documentation

➡️ `developer/`

Documentation intended for **developers and contributors**.

This section covers:
- building DyCoV from source,
- internal architecture and design decisions,
- extending or modifying the codebase.

End‑users who only want to run DyCoV do not need this section.

---

## Other documentation

The `docs/` directory may also contain additional subdirectories used for:
- reference material,
- figures and diagrams,
- tooling integration,
- documentation generation.

These are not part of the main user reading flow and are documented locally
where relevant.
